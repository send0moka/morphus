# Internationalisation and Localisation Notes

Morphus itself does not yet ship a translated UI. This document describes what
has been done to make the codebase internationalisation-ready and the plan for
adding locales.

## Current State

- All user-facing strings in the plugin UI are in English.
- Strings are **not yet extracted** into message catalogs; they are inline in
  the component files.
- The converter produces Figma node names in the language of the original HTML
  (pass-through; no translation).

## Internationalisation-Ready Patterns

Even before extraction, the codebase follows these conventions to make future
i18n easier:

1. **No string concatenation** for user messages: use template literals with
   named slots (`"Converted {count} nodes"`) rather than `"Converted " + count + " nodes"`.
2. **No hardcoded date/number formatting**: use `Intl.NumberFormat` and
   `Intl.DateTimeFormat` with a locale parameter.
3. **RTL awareness**: the plugin UI uses CSS logical properties
   (`margin-inline-start` instead of `margin-left`) where possible.

## Planned Localisation Workflow

1. Extract all user-facing strings into `src/plugin-ui/locales/en.json`.
2. Add a `useTranslation(key)` hook that falls back to `en.json`.
3. Ship additional locale files (`fr.json`, `de.json`, etc.) in the same directory.
4. Detect locale from `figma.currentUser.locale` at runtime.

## Contributing a Translation

Until the extraction is done, contributions are not accepted for translations.
Watch the issue tracker for the `i18n` milestone for updates.

Once extraction lands:

1. Copy `src/plugin-ui/locales/en.json`.
2. Rename it to `<locale>.json` (BCP 47, e.g. `fr-FR.json`).
3. Translate all values; do not translate the keys.
4. Open a PR with the new file only.
